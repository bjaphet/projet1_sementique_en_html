# Portfolio - Sémantique & Accessibilité (S2)

Bienvenue sur le dépôt de mon mini-projet de la Semaine 2. Ce projet a pour objectif de concevoir un portfolio personnel en utilisant une structure **HTML5 sémantique** stricte, sans recourir à des classes CSS personnalisées.

## 🎯 Objectifs du Projet
- **Architecture Sémantique :** Utiliser des balises explicites (`<header>`, `<main>`, `<nav>`, `<article>`, `<aside>`, `<footer>`) pour structurer le contenu.
- **Accessibilité (A11y) :** Concevoir un formulaire de contact conforme aux standards (usage de `<fieldset>`, `<legend>` et association `<label>`/`<input>`).
- **Défi "Classless" :** Respecter une structure HTML parfaite pour permettre au fichier `semanstyle.css` (fourni) de mettre en forme le site automatiquement.
- **Validation :** Garantir un code sans erreur selon le validateur W3C.

## 🛠️ Technologies utilisées
- HTML5 (Sémantique native)
- CSS3 (Feuille de style "Classless" pour la mise en page automatique)
- Outils : VS Code, Chrome DevTools, Validateur W3C

## 📋 Checklist de réalisation
- [x] Structure globale (header, main, footer).
- [x] Navigation via `<nav>`.
- [x] Utilisation pertinente des balises `<article>` et `<aside>`.
- [x] Hiérarchie des titres (h1 -> h2 -> h3) respectée.
- [x] Formulaire de contact 100% accessible.
